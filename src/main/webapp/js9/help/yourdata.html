Displaying Your Data With JS9

Loading a FITS File From Your Web Page

Once you have your Web page set up, you can make your FITS data files available on your JS9 Web page by creating URLs that call the JS9.Load() routine: 

    href='javascript:JS9.Load(filename, opts);'
where opts can be:
  • id: the id of a display (in the case of multiple instances of JS9)
  • imageOpts: an object containing params to be merged with the default JS9.imageOpts object

For example: 

    href='javascript:JS9.Load("fits/casa.fits");'
Clicking on this URL will cause the casa.fits file (here, stored in the fits directory) to be loaded into JS9. If the JS9 server-side helper is running, it will be alerted about this new file so that it can run analysis, etc.

To load an image into a specified display, pass the display id: 

    href='javascript:JS9.Load("fits/casa.fits", "myJS9");'

To override default image parameters, use the imgOpts argument: 

    href='javascript:JS9.Load("fits/m13.fits", {scale:"linear", colormap:"sls"});'

Note that the display can be passed in the imgOpts object along with the image parameter overrides: 

    href='javascript:JS9.Load("fits/3c273.fits", {display: "myJS9", scale: "linear"});'

Pre-Loading a File From Your Web Page

You can pre-load one or more files into JS9 when the Web page loads by add the JS9.Preload() routine to the onload event in your Web page body element: 

<body onload="JS9.Preload('fits/casa.fits', 
		          'fits/snr.fits', {colormap:'heat'},
		          'fits/3c58.fits',
		          'fits/m13.fits', {scale:'linear', colormap:'sls'},
		          'fits/3c273.fits',
		          'fits/ngc1316.fits', {scale:'linear'},
		          'fits/ngkper.fits', {colormap:'grey'});">
Note that this routine takes a list of images, each of which optionally can have the imgOpts (but not the simple display id) argument.
